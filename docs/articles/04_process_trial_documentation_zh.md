# 制程验证如何文档化

## 摘要

制程验证不是只记录结果，而是要记录目标、条件、观察、结果、分析与下一步计划。  
良好的制程验证文档可以让后续人员理解为什么做、怎么做、结果如何、下一步该做什么。

## 推荐结构

1. 验证目的；
2. 产品与规格；
3. 试验条件；
4. 数据来源；
5. 结果统计；
6. 外观与质量观察；
7. 工程分析；
8. 建议方向；
9. 下一步计划。

## 通用字段

| 字段 | 说明 |
|---|---|
| trial_id | 试验编号 |
| product | 产品 |
| target | 验证目标 |
| condition_group | 条件组 |
| result_summary | 结果摘要 |
| issue | 异常 |
| next_action | 下一步 |

## 为什么要模板化

制程验证如果没有模板，容易出现三个问题：

- 不同人员记录方式不同；
- 数据难以横向比较；
- 结论无法沉淀成组织知识。

## 开源价值

本项目提供通用模板，使制造业工程人员可以在不公开机密数据的情况下，建立可复用的验证记录方法。
