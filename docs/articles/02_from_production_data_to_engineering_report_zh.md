# 从生产数据到工程报告

## 摘要

制造现场的原始数据通常不适合直接给管理层或客户阅读。  
工程师需要把原始数据整理成结构化统计、趋势判断、异常说明与下一步行动。

## 常见问题

原始资料通常存在以下问题：

- 字段不统一；
- 产品名称不统一；
- 等级定义不统一；
- 数据来自多个 Excel 或截图；
- 备注文字缺少结构；
- 难以快速形成周报或正式报告。

## 推荐转换流程

1. 收集原始数据；
2. 统一字段；
3. 清理产品名称；
4. 按日期、产品、等级汇总；
5. 计算重量占比；
6. 标记异常项目；
7. 输出表格；
8. 转成技术报告。

## 工程报告建议结构

| 章节 | 内容 |
|---|---|
| 摘要 | 本期主要结论 |
| 产出统计 | 每日与产品别产出 |
| 质量分布 | A/B/C 等级比例 |
| 异常说明 | 主要异常与影响 |
| 工程分析 | 可能原因与关联 |
| 建议方向 | 下一步验证计划 |

## 开源实现

本项目中的 `production_summary.py`、`product_grade_statistics.py` 与模板文件可作为最小实现。

## 结论

数据本身不是报告。  
工程报告的价值在于把数据转成判断，把判断转成行动。
