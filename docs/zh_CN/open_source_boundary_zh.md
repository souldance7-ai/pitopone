# 开源边界说明

为了确保项目适合公开，本仓库采用以下边界原则。

## 可以公开的内容

| 类型 | 示例 |
|---|---|
| 通用字段 | date, product, grade, weight_kg |
| 合成数据 | 虚构产品、虚构重量、虚构日期 |
| 通用模板 | 周报、试验报告、质量报告 |
| 通用脚本 | 统计、汇总、字段检查、报告生成 |
| 方法论 | 如何做等级占比、如何结构化试验记录 |
| 技术文章 | 行业通用知识、非机密流程说明 |

## 不可以公开的内容

| 类型 | 示例 |
|---|---|
| 真实配方 | 单体比例、添加剂比例、专有路线 |
| 真实工艺窗口 | 温度、速度、张力、压力等内部参数 |
| 客户资料 | 客户名称、采购价格、验证反馈 |
| 公司资料 | 产能、成本、良率、商业策略 |
| 设备细节 | 未公开图纸、供应商专有设计 |
| 异常追溯 | 内部责任、人名、批次、索赔信息 |

## 推荐写法

把真实经验改写成：

- 通用化；
- 脱敏化；
- 方法化；
- 模板化；
- 合成数据化。

例如：

不推荐：

> 某客户某批次 100um 产品因某真实参数造成异常。

推荐：

> 在厚膜产品制程验证中，建议将产品别、厚度、等级、重量、观察项拆分为独立字段，以利后续质量趋势分析。
